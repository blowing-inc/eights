# Eights — User Guide

*The game of improbable battles*

---

## What is Eights?

Eights is a game tool for friends. You invent combatants — fictional characters, inside jokes, real people, abstract concepts — draft a roster, and battle them out round by round with your group voting on each result. The host makes the final call. Everything that happens is saved permanently: the combatants, the winners, the reactions, the chat, the story.

The goal is not just to win. It's to end up with a record of something that happened between real people that you can read back years later and still laugh.

## Getting Started

### Accounts vs. Guests

You can play as a guest by just entering a name. Guests work fine for a single session on one device.

If you create an account (tap the user icon in the top-right corner), your history, open lobbies, and combatants are tied to that identity. You can pick up a game from any device by logging in. Logged-in accounts are the preferred state — particularly for the host.

Guest sessions use browser storage and can be lost on a hard refresh or when switching between devices.

### Creating a Room

Tap "Create a room" on the home screen. You'll set your name and configure the room settings before the lobby opens.

**Room settings:**

| Setting | Description |
|---|---|
| Roster size (3–12, default 8) | How many combatants each player drafts. This determines the number of rounds — one round per combatant slot. |
| Allow spectators | People can join with a spectate link and watch without playing. They see rounds, chat, and reactions in real time but can't vote. |
| Anonymous combatants | Hides the owner's name on each combatant card during the voting round. Useful if you want voting to be purely on the combatant, not the player. |
| Blind voting | Individual votes are hidden until everyone has picked. Prevents pile-ons. The result is revealed all at once when the last vote is cast. |
| Bios required | Forces every player to write a short bio for each combatant before locking in their roster. Recommended if your group leans into the narrative. |
| Allow evolutions | When off, the Evolve option is hidden after wins. Winners are confirmed as-is. Evolutions are a fun ceremony — turn this off for faster games. |
| Allow draws | When off, the host can only confirm a winner each round — no draw option. |
| Allow merges | When draws are on, combatants that draw can merge into a new combined form. Turn this off to keep draws simple. |

Once the room is created, share the 4-character code or use the copy-link buttons to invite players and spectators.

### Joining a Room

Tap "Join a room" and enter the room code. If the game is already in progress, you may be offered a spectate option instead of joining as a player.

You can also join via a shared link, which will pre-fill the code.

## The Draft

Each player sees a private form with numbered slots. Fill in a name for each combatant. Bios are optional unless the host turned on "Bios required."

Your draft is auto-saved as you type. If you close the tab and return, your progress will be restored.

Lock in your roster when you're done. Other players' drafts are hidden — you don't see anyone else's combatants until the first round begins.

### Heritage Drafts (Series)

If this game is part of a series — a continuation of a previous game — your champions from the last game are required entries. You'll see your previous winners listed and must place each one in a slot before you can submit. They appear highlighted in your draft form.

If a champion evolved since the last game, their evolved form appears instead of the original. That variant carries the same owner and the same obligation.

### Force Start

The host sees a readiness tracker once they've submitted their own draft. If most players are ready, the host can force-start — any player who hasn't finished their draft won't have their combatants in the game. Use this when someone has gone quiet or the group is ready to move on.

## Rounds

The rounds screen shows the full round list as it fills up. Rounds are started by the host — tap the round button to advance.

Each round opens a voting screen for all players. One combatant per player is shown in the matchup. Players tap the combatant they think should win and submit their vote. The host sees all votes.

The host then:
- **Confirms a winner** — locks in the result and moves to the next round, or
- **Declares a draw** — both combatants are recorded as draws, no winner
- **Evolves the winner** — see the Evolution section below

The host can also undo the last round if a mistake was made, as long as the next round hasn't started.

### End Game Early

The host can end the game before all rounds are played. The game is marked as ended early. Combatants from that game won't be published to The Cast (since the game was incomplete).

### Game Complete

When all rounds are done, the host has three options:
- **Complete game** — marks the game as finished and returns home
- **View history** — see the full record of the game
- **Continue series** — start a new draft with the same players (see Series)

## Evolution

Evolution is the narrative heart of the game. When the host confirms a winner, they can choose to evolve that combatant instead — recording how this victory changed them.

### How it works

1. The host clicks "Evolve ⚡" on the winning combatant.
2. The host decides who writes the evolution — themselves, or they can hand it to the combatant's owner.
3. The writer sees a form with the combatant's current name pre-filled. They give the evolved form a new name. The bio field starts with the original bio as a baseline — they can rewrite it entirely or extend it.
4. On submit, the name is checked for uniqueness across The Cast. Evolution names must be novel — you can't evolve into something that already exists.

### What evolution creates

- A new entry in The Cast — the evolved variant is a distinct combatant with its own win/loss record, reactions, and bio
- A permanent evolution record on the round — storing who evolved from whom, what opponent caused it, and the game it happened in
- The original combatant is preserved exactly as it was. Its bio does not change. Its stats continue to accumulate in future games.

### What evolution does not do

- It does not replace the original in any other round of the current game. The draft roster is immutable — each combatant fights in its assigned slot.
- It does not affect the current game's remaining rounds.

### In the next game (series)

When a series continues, the evolved form appears in the owner's draft as a required pick — the owner must place the variant in a slot. The original is not required in its place; only the most recent evolved form is.

If a combatant evolved multiple times across multiple games, only the current tip form (the most recent variant) appears as the required entry.

If **Allow evolutions** is off, winners carry forward into the next series game as-is — no evolution prompt appears, and no variant is created. Champions still carry over as required picks.

### Reading the evolution in history

Every round that had an evolution shows "⚡ → [evolved name]" in the round list. The full round detail shows the evolution callout with the story: "[Original] evolved to [Variant] after the fight with [Opponent]."

## The Chronicles

Tap "The Chronicles" from the home screen to see all recorded games.

Games in a series are grouped together. Each group shows the total rounds played across the series and the date range.

### Game detail

Tap a game to open its full record:
- **Winners list** — a quick summary of every round result
- **Round navigator** — step through each round to see the combatants, votes, reactions, chat, and evolution record
- **Combatant roster** — all combatants from the game, sortable by player. Shows each combatant's result (W/L/D) and who they faced.

### Series standings

For a series (multiple connected games), the expanded series view shows standings: wins, losses, and total games per player across the whole series.

### Continue series

If you hosted a completed game, you'll see a "Continue series" callout in the game detail. Tapping it starts a new draft with the same players, carrying forward previous winners and evolved forms.

If you need to abandon a series after launching the next draft, you can "End series" from the draft screen. The completed games are preserved in history and the series can be resumed later.

## The Cast

The Cast is the permanent public record of every combatant ever entered across all games — once a game completes, its combatants are published here.

Each entry shows:
- Name and bio
- Win/loss/draw record
- Reactions (❤️ 😡 😂) accumulated across all games
- Owner
- For variants: the lineage — what they evolved from, who they beat, and which game it happened in

### Search

Use the search bar to find combatants by name, bio text, or player name.

### Sort

Sort by wins, losses, reaction counts, or alphabetically.

### View modes

- **Characters** — shows only base (generation-0) combatants. Each character appears once. Variants are surfaced inside the combatant's detail page.
- **All forms** — shows every published form including evolved variants.

### Combatant detail

Tap any combatant to see their full profile:
- Current name and bio
- Stats grid: wins, losses, draws, total rounds
- Head-to-head breakdown (expandable) — who they faced most and their record against each opponent
- Evolution lineage — for variants, the full story of every form they came from and what caused each evolution
- Bio history — previous versions of the bio if it was ever edited
- Reactions total

## Spectators

If the host allowed spectators, a separate share link is available in the lobby. Spectators join via that link and see:
- The lobby and player list
- The current round's combatants and matchup
- The round results as they are confirmed
- A running round history

Spectators cannot vote, react, or chat. They observe only.

## Host Controls Summary

| Phase | Controls |
|---|---|
| Lobby | Share room code/link, start game, cancel room |
| Draft | See readiness, force-start, cancel game / end series |
| Vote round | Confirm winner, declare draw, trigger evolution, undo last round |
| Rounds | Start next round, end game early, complete game, next game |
